# Survey Results: CRA Attestations and the Open Source Ecosystem

## Overview
This directory contains the results and analysis of the **Attestation Survey on the Cyber Resilience Act (CRA)**. 

The data collected here is a key asset of the **CRA Attestations project**, facilitating a deeper understanding of how the open source ecosystem can prepare for and implement the cybersecurity requirements mandated by the CRA. By gathering input from across the industry, we aim to publish clear guidance regarding **voluntary security attestations** (per Article 25), creating a sustainable pathway for open source projects to coexist with European market regulations.

## Why This Data Matters
The CRA introduces significant responsibilities for different actors in the digital supply chain:
* **Manufacturers:** Must ensure compliance for all components within a Product with Digital Elements (PwDE), including all integrated 2nd and 3rd party open source components.
* **Open Source Software Stewards:** Now face specific responsibilities regarding the management and communication of security vulnerabilities for the software they maintain with commercial intent.

The results of this survey provide the empirical evidence needed to lower the compliance burden on manufacturers by demonstrating how open source projects can provide standardized, high-quality security data.

## Data Transparency & Licensing
In keeping with the core values of the project:
* **Anonymity:** No personal or confidential information was collected during this exercise.
* **Open License:** All data and analysis in this repository are released under an **OSI-approved open source license**, ensuring the community can study, redistribute, and build upon these findings.
* **Replicability:** We have provided the raw data alongside our analysis to ensure that this exercise is fully replicable, reinforcing the value of embedding open source transparency into the traditional standardization process.
